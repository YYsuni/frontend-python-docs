## 一、两种错误：语法错误 vs 异常

### 1. 语法错误（SyntaxError）

- 代码**写得不对**，解析器直接报错，无法运行
- 常见：少冒号、括号不匹配、缩进错误

```python
# 错误：while 后少冒号
while True print('hello')
# 报错：SyntaxError: invalid syntax
```

### 2. 异常（Exception）

- 语法正确，但**运行时出错**
- 常见：除零、变量未定义、类型不匹配

```python
10 / 0          # ZeroDivisionError
print(spam)     # NameError
'2' + 2         # TypeError
```

## 二、异常处理：try / except / else / finally

### 核心流程

1. 先执行 `try` 代码块
2. 出错 → 跳匹配的 `except`
3. 没出错 → 执行 `else`
4. 无论如何 → 最后执行 `finally`

### 最简示例

```python
while True:
    try:
        x = int(input("输入数字："))
        break
    except ValueError:
        print("不是有效数字，重试")
```

### 多异常捕获

```python
try:
    f = open("test.txt")
    num = int(f.readline())
except OSError:
    print("文件打开失败")
except ValueError:
    print("转整数失败")
```

### 捕获异常信息

```python
try:
    1 / 0
except ZeroDivisionError as err:
    print("错误：", err)  # 输出：division by zero
```

### else：无异常才执行

```python
try:
    f = open("test.txt")
except OSError:
    print("打不开")
else:
    print("行数：", len(f.readlines()))
    f.close()
```

### finally：必执行（资源清理）

```python
def divide(x, y):
    try:
        return x / y
    except ZeroDivisionError:
        print("除零错误")
    finally:
        print("执行清理")

divide(2, 1)   # 正常 + finally
divide(2, 0)   # 报错 + finally
```

## 三、主动抛异常：raise

- 手动触发异常，用于**参数校验、业务报错**

```python
def set_age(age):
    if not isinstance(age, int):
        raise TypeError("年龄必须是整数")
    if age < 0:
        raise ValueError("年龄不能为负")

set_age(-5)  # 抛 ValueError
```

## 四、异常链：from

- 标记**异常因果关系**，方便定位根因

```python
try:
    open("no.txt")
except OSError as e:
    raise RuntimeError("打开数据库失败") from e
```

## 五、自定义异常

- 继承 `Exception`，命名以 `Error` 结尾
- 适合**业务专属错误**（前端类比：自定义错误码）

```python
class AuthError(Exception):
    """登录认证失败"""
    pass

def login(token):
    if not token:
        raise AuthError("token 不能为空")

try:
    login(None)
except AuthError as e:
    print("认证错误：", e)
```

## 六、自动资源清理：with 语句

- 替代 `try-finally`，**自动关闭文件/连接**（前端类比：自动释放）

```python
# 自动关闭文件，无需 f.close()
with open("test.txt", "r", encoding="utf-8") as f:
    content = f.read()
    print(content)
```

## 七、高级：批量异常（ExceptionGroup）

- 一次性抛出/处理**多个无关异常**

```python
def task():
    raise ExceptionGroup("批量错误", [
        OSError("文件错误"),
        ValueError("参数错误")
    ])

try:
    task()
except* OSError:
    print("处理OS错误")
except* ValueError:
    print("处理值错误")
```

## 八、最佳实践

1. **别裸 catch**：只捕获你能处理的异常，不 `except Exception` 通吃
2. **资源必清理**：文件/网络用 `with`，不用手动 `close`
3. **自定义异常**：业务错误用专属异常，便于分层处理
4. **异常信息**：用 `as err` 拿到详情，方便日志与排查
5. **finally 慎用 return**：会覆盖 try 里的 return
